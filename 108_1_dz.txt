Перед выполнением заданий важно самостоятельно повторить шаги, выполняемые на уроке.

1) Синхронизировать аппаратные часы с системными.
hwclock --adjust
hwclock -w
hwclock --localtime -w

2) Изменить часовой пояс.
cd /usr/share/zoneinfo
ln -sf /usr/share/zoneinfo/Europe/Moscow /etc/localtime; zdump /etc/localfile

3) Добавить в пул серверов времени российские сервера и синхронизироваться с ними.
ntp 0.ru.pool.ntp.org

# или через сервис ntp по инструкции ниже:
apt install ntp
nano /etc/ntp.conf
# Впиши:
server 0.ru.pool.ntp.org
server 1.ru.pool.ntp.org
server 2.ru.pool.ntp.org
server 3.ru.pool.ntp.org
# Выполни в тепминале:
systemctl reload ntp

4) Разобрать вывод команды ntpq -pn.
Покажет насколько и с чем мы синхронизируемся.Показывает север точного времени,у сервера времени показывает вышестоящий.Ntp использует несколько уровней серверов точного времени.Наверху-атомные часы, ниже-сервера, которые синхронизируются с атомными часами, затем сервера, которые синхронизируются с верхними серверами-те, что с атомными синхронизируются, далее находятся сервера, которым дан доступ из интернета для тех, кто хочет получить точное время.Также мы видим параметры какой сервер от какого отстаёт, задержку между серверами.Сервер выбранный для синхронизации обозначен * , +  сервер для обновления, - сервер который рекомендуется, х сервер который доступен.Параметры: delay - задержка времнни ответа от сервера.Ofset - разница между нашим сервером и сервером синхронизации.
